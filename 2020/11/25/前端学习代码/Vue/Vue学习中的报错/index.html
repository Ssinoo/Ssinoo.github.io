学习代码报错 收集

跨域

通过猫眼的NetWork中 xhr分类 一个link address中，vue用axios.get()方法

has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present on the requested resource.

跨域访问

查看NetWork中Header 的Access-Control-Allow-Origin

需要反向代理

在 vue.comfig.js中，Vue Cli中搜索 proxy

然后在 axios的get路径中，吧https://m.maoyan.com去掉，因为是要根据以/ajax开头的请求

改完vue.config.js要重新重启项目

另一种情况

有验证头

Access-Control-Allow-Origin: * 的时候

找到下面的

X-Client-Info 和 X-Host

X-Client-Info 和 X-Host 要以键值对来写

封装axios

因为请求 会有重复代码

  1. 创建文件夹

  2. 去官方文档 axios Creating an instance

    You can create a new instance of axios with a custom config.

    axios.create([config])

    3.然后使用自定义的axios

for in 的key值报错

Avoid using non-primitive value as key, use string/number value instead.

说明要绑定key值 绑定查到的ID值

axios 请求数据 调用报错

以为是异步的 ，先渲染了页面 再请求到了数据

数据设置为 null 的布尔为假 ，空数组，空对象 的布尔值为真

所以需要先判断，数据是否请求回来了
